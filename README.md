
# 🔄 RepoSync - Скрипт синхронизации репозиториев

**RepoSync** — набор скриптов для автоматической синхронизации локальных репозиториев с удалёнными и копирования их содержимого в другой репозиторий с заменой файлов, исключая `.git`. Скрипт работает как на **Linux/MacOS (Bash)**, так и на **Windows (PowerShell)**.  

---

## 🚀 Основные возможности

- 🔹 Интерактивный ввод ссылок на репозитории  
- 🔹 Синхронизация с удалённым репозиторием (`git pull`)  
- 🔹 Копирование содержимого в другой локальный репозиторий, исключая `.git`  
- 🔹 Безопасная замена файлов с одинаковыми именами  
- 🔹 Опциональный `git push` в локальном репозитории  
- 🔹 Совместимость с Linux, MacOS и Windows  
- 🔹 Тестирование через Docker, WSL или виртуальные машины  

---

## 📁 Структура проекта

| Файл | Описание |
|------|----------|
| `sync_repo.sh` | Bash скрипт для Linux/MacOS |
| `sync_repo.ps1` | PowerShell скрипт для Windows |
| `README.md` | Описание проекта и инструкции по использованию |

---

## ⚡ Как использовать

1. Клонируйте этот проект:
   ```bash
   git clone <ссылка_на_репозиторий>
   ````

2. Для **Linux/MacOS**:

   ```bash
   chmod +x sync_repo.sh
   ./sync_repo.sh
   ```
3. Для **Windows**:

   ```powershell
   .\sync_repo.ps1
   ```
4. Следуйте инструкциям скрипта:

   * Введите ссылку на ваш локальный репозиторий
   * Введите ссылку на репозиторий для синхронизации
   * Подтвердите опциональный `git push`

---

## 🔧 Примечания

* ❗ Никогда не используйте русские имена файлов и каталогов.
* ❗ Избегайте пробелов и спецсимволов в именах файлов и каталогов.
* 💡 Для тестирования используйте Docker, WSL или виртуальные машины.
* ⚠️ Всегда проверяйте пути, чтобы избежать потери данных.
